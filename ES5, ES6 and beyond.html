ES5, ES6 and beyond

What is ECMAScript?

ECMAscript is a standard.

It guarantees that when you write code following the rules of the standard it will run fine in a browser if the browser implements the same standard.

Why do we need a standard?

Back when JavaScript was first created, there was a war between browsers for market share. JavaScript was written for Netscape and Microsoft implemented their own version called JScript. This battle caused big problems for developers who had to write different scripts for different browsers.

To solve the problem, the language was standardized by ECMAscript International, and all browsers agreed to implement this new specification.
