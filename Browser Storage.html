Browser Storage

localStorage

The localStorage property allows you to access a local Storage object. Data stored in localStorage has no expiration time. Active tab can be refreshed, same destination can be opened in a new tab and data remains, and browser can be completely closed, but upon return to same destination, data remains.

.setItem()

When passed a key name and value, will add that key to the storage, or update that key's value if it already exists.

localStorage.setItem( 'username', 'Sample User');

.getItem()

When passed a key name, will return that key's value.

var savedUsername = localStorage .getItem('username') ;
console.log (savedUsername); // Sample User

.clearItem()

When invoked, will empty all keys out of the storage.

localStorage.clear( 'username');

sessionStorage

The sessionStorage property allows you to access a session Storage object for the current origin. Data stored in sessionStorage gets cleared when the page session ends. A page session lasts for as long as the browser is open and survives over page reloads and restores. Opening a page in a new tab or window will cause a new session to be initiated.

.setItem()

When passed a key name and value, will add that key to the storage, or update that key's value if it already exists.

sessionStorage.setItem( 'tempUsername', 'Sample User');

.getItem()

When passed a key name, will return that key's value.

var savedTempUsername = sessionStorage .getItem('tempUsername') ;
console.log (savedTempUsername); // Sample User

.clearItem()

When invoked, will empty all keys out of the storage.

sessionStorage.clear( 'tempUsername');
