Var, Let & Const

JavaScript variables are containers for storing data values.

However, as JavaScript has matured, new options have been added for storing your data. Each of these options offers slightly different advantages and disadvantages.

var

The var keyword has function scope, meaning (depending on where it is created) it is accessible everywhere. Good? Yeah.

var coffee = 'Dutch Bros' ;
function myCoffee() {
console.log (coffee); // Dutch Bros
}
myCoffee();
console.log (coffee); // Dutch Bros

Note: Here we created a variable outside of all functions making it a "global" variable.

It can be instantiated and instantiated again without error, and changed at any time into any type of data.
Good? Meh.

var coffee = 'Dutch Bros' ;
function myCoffee() {
var coffee = 'Starbucks' ;
console.log (coffee); // Starbucks
}
myCoffee();

Note: Here we created a variable globally, AND one inside of myFunction which is only accessible INSIDE of myFunction.

var coffee = 'Dutch Bros' ;
function myCoffee() {
var coffee = 'Starbucks' ;
console.log (coffee); // Starbucks
coffee = 'Peets';
console.log (coffee); // Peets
}
myCoffee();
console.log (coffee); // Dutch Bros

Note: In this example the "scoped" variable we created inside of myFunction and logged it. Is modified

It can even be instantiated without the keyword.
Good? Nope.

function myCoffee() {
coffee = 'Dunkin';
console.log (coffee); // Dunkin
}
myCoffee();
console.log (coffee); // Dunkin

Because variables have function scope, instances such as the example below can cause a problem if you're not careful.

for(var i = 0; i < 5 ; i++){
console.log (i); // 0 1 2 3 4
}
console.log (i); // 5

let

The let statement declares a block scope local variable, optionally initializing it to a value..

for(let i = 0; i < 5 ; i++){
console.log (i); // 0 1 2 3 4
}
console.log (i); // ReferenceError: i is not defined
let fName;
console.log (fName); // Undefined
fName = 'Matt';
console.log (fName); // Matt
fName = 'Matthew';
console.log (fName); // Matthew
`let` works very similarly to `var`, however, because it is block scope, values donâ€™t bleed out from the code block. You also, can not redeclare a variable as you can variables.
let fName = 'Matt' ;
console.log (fName); // Matt
let fName = 'Matthew' ;
console.log (fName); // SyntaxError: redeclaration of let fName

const

The strictest of all the statements. It declares a read-only named constant. It shares the same scope as let, however it must me initialized when the const is declared.

const fName;
console.log (fName); // SyntaxError: missing = in const declaration
