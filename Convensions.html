Capitalization

Good:

var camelcase;
var camelCase;
var camel_case;
var _camelcase;
var $camelcase;

Not technically bad, but we won't do this:

var CamelCase;

Bad:

var camel case;
var camel-case ;
var 1camelcase;

Indention

Although this is merely a recommendation, it comes from personal experience. Indent your code. HTML, CSS, JavaScript, everything. When you run into a problem, it will be exponentially easier to find the problem if you properly intent your code.

Reserved Words

There are some words, sometimes called keywords that are reserved in the JavaScript language. These words have special meaning and functionality. Here is a short list of the ones that we’ll come across:

Most Common if/else if/else loops switch statements Other useful keywords
var if for switch let
function else if while case const
return else do continue typeof
this in break
new

Tip: You probably won’t find on any list a reserved word:

  • name

However, it IS utilized by the window object, so don’t use it. It'll cause problems. :)
