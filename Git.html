Git

Basic Commands

git status
git add .
git commit -m "your commit message"
git push

Creating a Repository

cd ~/Desktop
mkdir geekwise-javascript1
cd geekwise-javascript1
touch README.md .gitignore
git init

Notice on line 4, we're actually creating two files in one line. The first: README.md is there to provide information or instruction to the code in your repository.

The second: .gitignore is a little more important. We'll get into the details towards the end of the course, but for now, go to this address:

https://goo.gl/gDm3wC

Copy and all the code from the Gist window and paste it into your new .gitignore file. We however need to add a couple more lines to the very end of the file:

# Environment Variables #
#########################
env.json

Github

From http://github.com create a new repository.


After you click Create repository, you'll see this rather disconcerting page, but worry not, there are only a few lines that we need to concern ourselves with.


Under the headline '...or create a new repository on the command line' we're going to pick up where we left off.

git add .
git commit -m 'first commit'
git remote add origin git@github.com:[YOUR USERNAME]/[YOUR REPOSITORY].git
git push -u origin master

Don't worry... the next time you add new content the commands are simpler. Github just needed some additional options the first time around. See the top of the page under Basic Commands.
