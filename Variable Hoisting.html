Variable Hoisting

Hoisting is JavaScript's default behavior of moving all declarations to the top of the current scope (to the top of the current script or the current function).

var x = 1 ;
console.log (x);

var x declares the variable x, = 1 initializes (or assigns it) a value. What will happen if we tried to console.log before the variable has been declared and assigned?

console.log (x);
var x = 1 ;

Did it work or not work as you expected? Why?

As mentioned earlier, declarations are hoisted to the top of the scope.

Note: Hoisting (as far as variables is concerned) is only an "issue" with the var variable. Let and const do NOT get hoisted!

var x = 1 ;
let y = 2 ;
const z = 3 ;
console.log (x);
console.log (y);
console.log (z);
console.log (a);
console.log (b);
console.log (c);
var a = 1 ;
let b = 2 ; // ReferenceError: b is not defined
const c = 3 ; // ReferenceError: c is not defined

Here's another example:

var x = 1 ; // Initialize x with the value of 1
console.log (x + " " + y) ; // 1 undefined
var y = 2 ; // <- the variable name is saved to memory, not the value!
// The above code is the same as below
var x = 1 ; // Initialize x with the value of 1
var y; // Declare y
console.log (x + " " + y) ; // 1 undefined
y = 2; // Initialize y with the value of 2

It is a best practice to always declare your variables at the start of your document.
